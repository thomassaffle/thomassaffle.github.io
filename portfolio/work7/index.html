Fragment - Project Management Application

Interaction Design / Visual Design

In 2018 I, and six others, worked as a team to create a project management tool we called Fragment, which is a light-weighted web based application. Each one of us had bad experiences with project management tools daily, and we knew there were many people who felt the same way too. Our team wanted to find a new and easier way to improve the project management experience.

Our goal for this application was to make an easy to use project management tool for freelancers, startups, and small to medium creative companies who want budget project management.

I worked as the UX/UI designer on first generation of this project. I helped define the basic narrative of this app, I led UX design process, and I was responsible for the visual and interface design for this project.

Initial user research and creating personas

We sent out surveys and conducted interviews to gather info on user behavior and their expectations for a project management tool. In total, we got 54 responses. After analyzing this user feedback, we had a brainstorm session for mapping possible use cases. With the use cases in place, we were able to define detailed personas.

User Stories and User Journeys

Based on our personas’ needs and our business goals we created 5 main user stories. For each story, we mapped the user journey. User journey mapping helped us break down each main user story into tasks later.

User stories:

  • Jack wants to easily manage multiple projects.
  • Jack wants to plan new projects quickly and easily.
  • Scott wants to communicate more easily with Fragment.
  • Joe wants to share her designs instantly with everybody on the project team.
  • Will wants to update his development progress faster.

Experience maps

I structured the whole experience map with low-fidelity prototypes so we could have a general idea of the size and complexity of this app.

Creating wireframes

Then we focused one each page to develop space allocation, prioritization of content, functionality, and intended behaviors. We followed the “mobile first” concept to create wireframes for small screens first, then for larger screens. Determining the information hierarchy of the design helped us plan out the content and user experience.

High-fidelity design

I created high-fidelity prototypes based on our design style guide with real content. This guided our developers during the building process. We also used these to build interactive prototypes using Invision to help us with user testing.
