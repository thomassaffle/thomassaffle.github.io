University of Arizona Library - Homepage Redesign

Website Design / Information Architecture / Visual Design

In 2018, the University of Arizona Libraries decided to move their main site from Drupal 7 to Drupal 8. Homepage redesign was a sub-project of this migration. I served as a UX/UI designer in this project, responsible for delivering design, also participated in user research, information architecture decisions.

Project Goals

  • Increase user experience by creating intuitive and functional interface.
  • Maximize library impact with clean and modern responsive design.
  • Keep current with UA marketing strategy through application of new UA branding.

Before and After Effect

Main Navigation

To update the current main navigation bar, I suggested to change the narrow drop-down menu to a full width drop-down menu, making it much easier to track targets. With regards to responsive design, the new header has two break points. This leads to 3 layouts for large, medium, and small screens. For large screen, I added a short explanation to each category heading, and I added related link function providing short cuts to popular content pages under each category. For medium screen, I also used the full-width drop down menu, however, instead only viewing one category at a time it contains all categories. For small screen, I used the typical collapsed drop-down design.

Library Status Panel

In the new design, besides providing library hours as the old homepage did, I added an indicator icon so users can tell more easily if any one library is open or closed. We also provided a link to Google Maps for library locations. The library’s study-room booking and technology-lending systems both started using a single application called Libcal, which makes it easy to access real-time data identifying how many study rooms and tech devices are currently available. I added this information to this panel so that users can more easily check whether study rooms or devices are available.

Header

The current header’s height is fixed to 200px, take too much assets especially for small screens. In the new design I reduced the height by about half even as we retained the content and functions of the old version.
